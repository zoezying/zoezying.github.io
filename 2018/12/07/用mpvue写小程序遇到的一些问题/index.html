用 mpvue 框架开发小程序遇到的一些问题

发表于 2018-12-07   |   分类于 mpvue   |  

摘要:
用 mpvue 框架开发小程序遇到的一些问题的总结。

created 生命周期问题

mpvue 的生命周期 created 跟 vue 的生命周期 created 的不一样，后者是进入当前页面的时候才会触发，而前者是不管有没有进入当前页面都会触发 created 生命周期。所以不推荐在 created 生命周期做数据的初始化，可以在小程序自带的 onLoad 生命周期进行数据的初始化操作。

数组遍历中出现 should hava explicit keys 错误

在使用 v-for 进行遍历的时候，必须要设置 key ，否则会一直报错。

还有在嵌套列表渲染的时候，必须指定不同的索引，否则会一直报错。

退出页面重新进入，页面数据未初始化

需求是：主页 -> A 商品页面 -> 商家页面 -> B 商品页面。

但是从商家页面进入 B 商品页面的时候，页面数据依然是A商品的。此情况出现的原因可能是 mpvue 的混合周期留下的问题。

自定义的组件不支持添加 class

自定义的组件不支持添加样式名，否则会报错，可以套个父标签，把样式名添加到父标签上。

textarea 组件文字穿透问题

由于 textarea 组件是小程序底层组件，层级比较高，又无法给它设置层级，所以会导致覆盖在它上面的元素，会出现文字穿透问题。

解决方法：
可以先将 textarea 隐藏掉，然后添加个跟 textarea 组件相同样式的标签，用来向用户展示，然后给该标签添加点击事件，触发点击事件就将 textarea 组件显示出来，并且隐藏自身。然后给 textarea 组件绑定 bindblur 事件，当用户输入完毕，失去焦点之后，就将展示用的标签显示出来，并且赋值给它展示，然后隐藏自身。

注意: 需要设置 textarea 组件的自动聚焦 auto-focus ，否则用户需要点击两次才能进行输入。

相对路径的图片不显示

使用相对路径的图片将不会显示，图片的引入只能使用绝对路径，或者远程图片路径才能显示。

新增 page 页面，使用微信开发者工具出现报错

webpack编译文件是由entery决定的，新增的页面没有加入到entery。

解决办法：重新 npm run dev
